Desarrollo de FETCH-JSON

La Fetch API es una versión más simple y fácil de usar para consumir recursos de forma asíncrona que una XMLHttpRequest . Fetch te permite trabajar con REST APIs con opciones adicionales, como almacenar datos en caché, leer respuestas de transmisión y más. La diferencia principal es que Fetch funciona con promesas, no con devoluciones de llamada. Los desarrolladores de JavaScript se han alejado de las devoluciones de llamadas después de la introducción de las promesas.

FETCH-JSON





"fetch" es una función incorporada en JavaScript que se utiliza para realizar solicitudes de red y obtener recursos de forma asíncrona. Permite realizar peticiones HTTP y obtener datos de servidores web. La función fetch devuelve una promesa que se resuelve con la respuesta de la solicitud.

El formato JSON se basa en pares clave-valor y está compuesto por objetos y matrices. Los objetos se delimitan con llaves "{" y "}", y los elementos dentro de un objeto se separan con comas. Cada elemento tiene una clave y un valor separados por dos puntos ":". Las matrices se delimitan con corchetes "[" y "]" y los elementos dentro de una matriz también se separan con comas.
En resumen, "fetch" es una función de JavaScript utilizada para realizar solicitudes de red y obtener recursos de forma asíncrona, mientras que "JSON" es un formato de intercambio de datos basado en texto ampliamente utilizado en el desarrollo web. La combinación de "fetch" y "JSON" se utiliza comúnmente para obtener y trabajar con datos JSON de manera asíncrona en una aplicación web.
